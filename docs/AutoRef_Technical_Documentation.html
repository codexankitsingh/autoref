AutoRef вҖ” Technical Documentation

1. Project Overview

AutoRef is a full-stack SaaS platform that automates cold outreach for job seekers. It uses LLM-powered email generation (Google Gemini) to semantically match resumes against job descriptions, generates personalized referral emails, sends them via Gmail OAuth, and orchestrates automated follow-up sequences вҖ” all from a single dashboard.

Key Metrics: - Reduces cold email drafting from ~15 minutes to ~45 seconds (95% reduction) - Supports 3 target role categories with dedicated resume matching - Automated 3-stage follow-up pipeline with reply detection - Multi-tenant architecture with admin approval flow

3.2 JWT Token Flow

Token Lifetime Purpose
Access Token 24 hours API authentication via Authorization: Bearer <token>
Refresh Token 7 days Obtain new access token without re-login

Token payload:

{
  "sub": "42",        // user_id as string
  "exp": 1716000000,  // expiry timestamp
  "type": "access"    // or "refresh"
}

Why HS256 (symmetric) over RS256 (asymmetric)? - Single backend server вҖ” no need for public key distribution - Simpler config (one secret vs key pair) - Lower computational overhead per request

3.3 Password Security

  • Algorithm: bcrypt via passlib
  • Why bcrypt over argon2? Wider library support, battle-tested, sufficient for this scale
  • Passwords are never stored in plaintext вҖ” only the bcrypt hash
  • password_hash is nullable to support Google OAuth-only accounts

3.6 Gmail OAuth (Mail Connection) вҖ” State Parameter Fix

Problem: Gmail OAuth callback is a redirect from Google вҖ” no Bearer token attached.

Solution: We pass user_id in the OAuth state parameter: 1. GET /api/auth/gmail вҶ’ passes state=current_user.id to Google 2. Google redirects back to /api/auth/gmail/callback?code=xxx&state=42 3. Callback reads state вҶ’ looks up user by ID вҶ’ links Gmail account

Scope relaxation: We set OAUTHLIB_RELAX_TOKEN_SCOPE=1 because Google sometimes returns scope strings in a different order than requested, causing the library to throw вҖңScope has changedвҖқ errors.

11. Design Trade-offs & Interview Defense

Q: вҖңWhy SQLite instead of PostgreSQL?вҖқ

A: Operational simplicity. SQLite is zero-config, file-based, and free. For a single-tenant SaaS with <10K rows and no concurrent write pressure (one user at a time), SQLite outperforms managed Postgres on cost ($0 vs $7+/month) and latency (no network hop). Migration path to Postgres is trivial вҖ” SQLAlchemy abstracts the dialect.

Q: вҖңWhy not use Celery for background jobs?вҖқ

A: APScheduler runs in-process with no external broker dependency. For a single-server deployment sending ~60 emails/hour max, the overhead of Redis + Celery worker processes is unjustified. APSchedulerвҖҷs BackgroundScheduler gives us cron-like scheduling with 5 lines of code.

Q: вҖңWhy JWT over session cookies?вҖқ

A: Decoupled frontend (Vercel) and backend (Render) вҖ” different domains. Session cookies require SameSite=None + complex CORS. JWTs in Authorization headers work cleanly across origins. Trade-off: JWTs canвҖҷt be server-side revoked without a blacklist; we accept this since tokens expire in 24h.

Q: вҖңWhy localStorage for tokens?вҖқ

A: Simpler than httpOnly cookies for a cross-origin SPA. XSS risk is mitigated by: (1) Next.js auto-escapes JSX, (2) no dangerouslySetInnerHTML on user input, (3) CSP headers on Vercel. For a portfolio project, this is an acceptable trade-off over the complexity of a BFF (Backend-For-Frontend) cookie proxy.

Q: вҖңHow do you prevent abuse?вҖқ

A: Three layers: 1. Admin approval: New users canвҖҷt access the app until manually approved 2. Rate throttling: Follow-ups limited to 1/minute via APSchedulerвҖҷs steady drip 3. Scope isolation: Users only see/modify their own data (row-level filtering on user_id)

Q: вҖңWhy prompt engineering over fine-tuning for email generation?вҖқ

A: Fine-tuning requires curated training data (hundreds of labeled email pairs) and ongoing model management. Prompt engineering with gemini-2.5-flash-lite gives us 90% of the quality at 0% of the training cost. The role-specific role_configs dict acts as a lightweight вҖңsoft fine-tuneвҖқ вҖ” it injects domain vocabulary and category guidance without model modification.

Q: вҖңWhat happens if the LLM hallucinates?вҖқ

A: Three-layer defense: 1. Prompt rules: вҖңDO NOT hallucinate projects, metrics, or experiences I do not haveвҖқ 2. Factual grounding: UserвҖҷs profile_text is injected as the only source of truth 3. Post-processing regex: Catch-all \[.*?\] strips any remaining placeholder artifacts
